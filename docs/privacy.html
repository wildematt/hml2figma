Html2Figma Privacy Policy

Last updated: March 8, 2026

This Privacy Policy explains how the Html2Figma Chrome extension handles user data.

1. What the extension does

Html2Figma helps users capture webpage structure and visual information, then convert it into content that can be used in Figma.

2. Data we process

The extension may process the following data when the user actively runs capture:

  • Current page URL and page title.
  • DOM structure and computed style information needed for conversion.
  • Image/video resource references required for rendering fidelity.
  • User-selected element selector when user chooses partial capture.

3. How data is used

  • Only to provide the core feature: converting webpage content for Figma workflows.
  • Only after user action (for example: clicking the extension button or capture options).

4. Data sharing and transfer

  • The extension does not sell personal data.
  • If a user chooses flows that involve sending data to Figma-related endpoints, the captured content is transferred only for processing requested by that user.

5. Data storage and retention

  • The extension itself does not maintain a long-term local database of captured page content by default.
  • Any temporary in-memory data exists only during capture/conversion flow.

6. Permissions rationale

  • activeTab: used to access the currently active tab only when user triggers the extension.
  • scripting: used to inject capture scripts required for page-to-Figma conversion.

7. Your choices

  • You can stop data processing by not triggering capture actions.
  • You can disable or uninstall the extension at any time via Chrome extension settings.
